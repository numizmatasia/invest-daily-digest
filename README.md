# Личный инвестиционный помощник v2

Проект делает ежедневный инвестиционный отчет в Telegram. Цель — не пересказывать новости, а помогать принять решение: покупать, продавать, ждать или ничего не делать.

## Текущий статус

Версия проекта: v1.0 рабочая база.

Замороженное ядро:
- `main.py`
- `decision_engine.py`
- `portfolio.json`
- `watchlist.json`
- `cash_log.json`

Эти файлы не менять без отдельного решения владельца проекта.

## Как работает проект

Схема:

```text
RSS-источники
↓
news_processor.py
↓
event_engine.py
↓
decision_engine.py
↓
Gemini
↓
Telegram
```

## Что делает каждый файл

### `main.py`
Главный файл запуска. Он:
- читает RSS;
- загружает портфели и watch list;
- запускает обработку новостей;
- передает данные в движок решений;
- отправляет результат в Telegram.

### `news_processor.py`
Очищает новости:
- убирает HTML;
- чистит текст;
- удаляет дубли;
- готовит новости для анализа.

### `event_engine.py`
Превращает новости в инвестиционные события:
- ИИ и энергопотребление;
- уран и ядерная энергетика;
- нефть и энергетика;
- крипто;
- полупроводники;
- металлы;
- общий рынок;
- потенциальные новые идеи.

### `decision_engine.py`
Главный движок решений. Он:
- учитывает размер позиций;
- разделяет Freedom и Paidax;
- показывает только активы, требующие внимания;
- формирует решение дня;
- рассчитывает сумму покупки;
- выводит блоки Watch List и спекулятивных идей.

### `portfolio.json`
Единственный источник правды по текущим портфелям.

Сейчас содержит:
- Freedom;
- Paidax.

Все значения указаны как доля позиции в процентах.

### `watchlist.json`
Список бумаг для наблюдения:
- AVGO;
- AMZN;
- GOOGL;
- SKHY / SK Hynix;
- META.

### `cash_log.json`
Учет инвестиционного бюджета:
- минимальный месячный план пополнения;
- внесено;
- инвестировано;
- остаток свободного кэша.

### `ARCHITECTURE.md`
Краткое описание утвержденной архитектуры.

### `.github/workflows/daily.yml`
Автоматический запуск через GitHub Actions.

### `requirements.txt`
Список Python-библиотек.

## Как запустить вручную

1. Открыть GitHub-репозиторий.
2. Перейти во вкладку **Actions**.
3. Выбрать workflow **Daily Digest**.
4. Нажать **Run workflow**.
5. Дождаться Telegram-сообщения.

## Секреты GitHub

В настройках репозитория должны быть заданы:

```text
TELEGRAM_BOT_TOKEN
TELEGRAM_CHAT_ID
GEMINI_API_KEY
```

Без них бот не отправит сообщение.

## Что можно менять вручную

Можно менять только данные:

- `portfolio.json` — если изменился состав или доли портфеля;
- `watchlist.json` — если меняется список наблюдения;
- `cash_log.json` — если были пополнения или покупки.

Кодовые файлы без необходимости не трогать.

## Что нельзя делать без отдельного решения

Нельзя без отдельного решения менять:

- архитектуру проекта;
- `main.py`;
- `decision_engine.py`;
- список RSS-источников;
- формат Telegram-отчета;
- структуру портфелей.

## Как обновлять портфель

Если куплена или продана бумага, нужно обновить `portfolio.json`.

Пример:

```json
"Paidax": {
  "AMAT": 31.66,
  "PL": 21.33,
  "MU": 17.14,
  "NVDA": 11.25,
  "SPYG": 7.45,
  "VT": 7.23,
  "GLD": 2.19,
  "VRT": 1.74
}
```

Доли должны суммарно давать около 100% внутри каждого портфеля.

## Как обновлять cash_log.json

Пример текущего месяца:

```json
{
  "2026-07": {
    "target_min_usd": 400,
    "deposited_usd": 0,
    "invested_usd": 0,
    "cash_left_usd": 0
  }
}
```

Если внесено 400 $, но ничего не куплено:

```json
{
  "2026-07": {
    "target_min_usd": 400,
    "deposited_usd": 400,
    "invested_usd": 0,
    "cash_left_usd": 400
  }
}
```

Если внесено 400 $ и куплено на 150 $:

```json
{
  "2026-07": {
    "target_min_usd": 400,
    "deposited_usd": 400,
    "invested_usd": 150,
    "cash_left_usd": 250
  }
}
```

## Формат Telegram-отчета

Отчет должен начинаться с решения:

```text
📌 РЕШЕНИЕ НА СЕГОДНЯ
⏳ Ждать / ✅ Купить / 💰 Продать

Сумма покупки сегодня:
0 $
```

Дальше:
- инвестиционный бюджет;
- Freedom;
- Paidax;
- Watch List;
- спекулятивные идеи;
- что сегодня не делать;
- план действий.

## Главные правила проекта

1. Решение важнее новости.
2. Лучше ничего не делать, чем сделать плохую сделку.
3. Если позиция уже крупная, не наращивать без сильной просадки.
4. Gemini объясняет, но не принимает решение.
5. Портфели Freedom и Paidax не смешивать.
6. Тикеры показывать только если они требуют внимания.
7. Не усложнять архитектуру без необходимости.
